## Writing Notes
- 我默认用第一人称记录这一天的工作，语气可以带一点日记感，但要保持结构化、可检索。
- 每条记录优先写清楚：topic / context / actions / decisions / validation / artifacts / next。

## Entry Schema
- topic: 本轮主主题
- context: 触发背景或问题来源
- actions: 本轮实际动作
- decisions: 对后续有影响的约束或结论
- validation: 已做检查与结果
- artifacts: 关键文件或目录
- next: 后续待跟进事项
